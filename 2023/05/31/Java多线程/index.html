2. 同步方法和同步块

  • 将一个方法上加上synchronized关键字即可将方法声明为同步方法

  • 同步方法会为当前方法所在对象加锁，即this。不能将一个大方法声明为synchronized，会影响效率

  • 同步块可以在括号中加入要加锁的对象来局部加锁。

线程安全的集合
  • 数组：CopyOnWriteArrayList<T>
  • Map：ConcurrentHashMap
锁

  • 死锁：多个线程各自占有一些共享资源，并且互相等待其他线程占有的资源才能运行，而导致两个或者多个线程都在等待对方释放资源。（如果一个同步块中同时持有两个以上对象的锁，就可能发生死锁的问题）

方法名 作用
wait() 表示线程一直等待，直到其他线程通知，与sleep不同，会释放锁
wait(long timeout) 指定等待的毫秒数

notifyAll() 唤醒同一个对象上所有调用wait()方法的线程，优先级别高的线程优先调度
