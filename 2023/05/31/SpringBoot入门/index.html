基本使用

  • 目的：用来简化Spring原生开发
  • 优点：简化了Spring原生开发中初步搭建过程和开发过程，且最后不需要额外提供tomcat服务端而是直接运行
  • 原理：使用起步依赖方法帮助开发者导入若干配置
  • 基本操作：
    • 创建boot程序和controller类
    • 运行Application.class

  1. SpringBoot配置文件方法：
    • ① 在resource下的application.properties中配置（优先的配置）
    • ② 在application.yml中配置（推荐）
    • ③ 在application.yaml中配置（不推荐因为idea不提示，可以在模块配置的Fecets中加上配置解决）

yaml格式

  • 优点：易阅读，重数据，轻格式
  • 规则：
    • 大小写敏感
    • 数据前有一个空格
    • 不同层级用空格区分
读取自定义配置

  • 在boot程序中可以通过：

    • 注入的形式读取配置（如：”${user.name}”，”${result.data.array[0]}”）

    • 使用自动装配的Environment类来自动读取配置

    • 使用类来接收

      • @Component
@ConfigurationProperties(prefix = "user")
public class User {
}

SpringBoot整合Mybatis

  • 区别：

    • 无需手写MybatisConfig和JdbcConfig，SpringBoot将自动生成
    • 无需手动导入properties，只需要在SpringBoot配置中写上dataSource有关参数
    • 无需手动扫描mapper，只需要在原来mapper类上加入@Mapper注解，SpringBoot将自动扫描该类

  • 注意：

    • 生成项目时应加入Mybatis和mysql的起步依赖
    • 如需要更改数据源类型，只需要导入依赖并在dataSource配置中加上type参数即可
